# Task6-Sales_Trend_Analysis Using Aggregations
SQL project analyzing monthly sales trends using aggregate functions, date functions, GROUP BY, ORDER BY, WHERE, and LIMIT in SQL

## Objective
Analyze monthly revenue and order volume from sales data using SQL aggregate functions.

## Tools Used
- MySQL Workbench
- SQL

## Dataset
- Superstore Sales Dataset
- Table Name: `orders`

## Concepts Covered
- EXTRACT()
- STR_TO_DATE()
- SUM()
- COUNT(DISTINCT)
- GROUP BY
- ORDER BY
- WHERE
- LIMIT
- ROUND()

## SQL Queries Performed

### 1. Monthly Revenue and Order Volume
Calculated total monthly revenue and total number of unique orders using aggregate functions.

### 2. Sales Analysis for the Year 2017
Filtered records for the year 2017 and analyzed monthly revenue and order volume.

### 3. Top 3 Months by Revenue
Identified the top three months with the highest revenue using ORDER BY and LIMIT.

## Output
- Monthly Order Volume Analysis
- Revenue Trend for 2017
- Top 3 Revenue-Generating Months
